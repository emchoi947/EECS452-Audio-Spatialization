To compile the receiver_tag code, I used the ESP IDF extension in VSCode. I used the Windows version rather than the WSL connection as WSL cannot sense COM ports without going through a process with the usbipd-win tool to add the COM ports to WSL. I flashed the code to the ESP32-H2 by setting the target to an ESP32-H2, specified the COM port as to which the ESP32-H2 was connected via USB-C, and built and flashed the code using the ESP IDF interface. After flashing, the code should run immediately on the ESP32-H2 and continue to run as long as there is power.

To compile the spatialization_algorithm.PDE code you need to utilize the Teensy 4.1 board manager extension in Arduino. This allowed me to use the Arduino IDE directly when flashing and uploading onto the Teensy, which made serial monitoring much easier for testing behaviors. You need to flash the code to the Teensy 4.1 through the port selection on the Arduino IDE, whichever port you have connected to the micro-USB input on the Teensy. Then you build and flash the code using the run/upload button on the Arduino IDE, and it should run as long as the system has power.
